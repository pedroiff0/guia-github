# guia-github

Repositório público com boas práticas, templates e exemplos práticos para **GitHub Actions**, **Issues**, **Projects** e **Branches**, com foco em projetos Python.

## Objetivo
Criar um guia completo e reutilizável para equipes que queiram padronizar processos no GitHub: CI/CD, templates de issue/PR, políticas de branch, automações de Projects e exemplos "in-loco".

## Licença
MIT — veja `LICENSE`.

## Como usar
- Leia `docs/*.md` para orientações por tema.
- Use os templates em `.github/` para padronizar issues e PRs.
- Experimente os workflows em `.github/workflows` diretamente no GitHub (há exemplos acionáveis).
