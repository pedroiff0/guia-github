# Git & GitHub Best Practices

- Use Conventional Commits (feat, fix, docs, chore) para histórico claro.
- PRs pequenas e atômicas; explique o porquê das mudanças.
- Adote `CODEOWNERS` e políticas de revisão (ex: 1-2 reviews required).
- Automatize manutenção: Dependabot, stale bot e labeler.
- Mantenha um CHANGELOG (Keep a Changelog) e use releases semver.
